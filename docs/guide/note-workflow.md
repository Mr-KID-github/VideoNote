# 笔记工作流

## 生命周期

1. 客户端提交生成请求
2. 后端在 `output/` 下创建任务目录
3. 下载媒体或准备输入
4. 按配置的转写器执行转写
5. 调用 LLM 生成 Markdown
6. 注入关键时刻、时间戳与截图占位符
7. 持久化任务产物并更新状态
8. 前端将最终笔记保存到数据库

## 总结模式

- `default`
  - 小内容走一次性总结，长内容自动升级为分层总结
- `accurate`
  - 总是先分块总结再合并
- `oneshot`
  - 总是一次性发送完整转写文本

## 用户体验上的结果

- 笔记标题可带时间戳跳转
- 关键时刻可以附带截图
- 预览侧媒体可跟随时间戳 seek
- 保存后的笔记可以生成公开只读分享链接
