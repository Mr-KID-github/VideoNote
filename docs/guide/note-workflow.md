# 笔记工作流

## 输入模式

- 视频 URL：后端负责下载远程媒体，再进入转写和总结流程
- 本地音频 / 视频：浏览器通过 `multipart/form-data` 上传媒体文件，再进入转写和总结流程
- 本地文字稿：浏览器上传 `TXT`、`MD`、`SRT`、`VTT`、`JSON`，后端解析后直接跳过 STT

## 生命周期

1. 客户端提交 URL、本地音视频或文字稿生成请求
2. 后端在 `output/` 下创建任务目录
3. 下载远程媒体、保存上传文件，或解析上传文字稿
4. 媒体任务按配置的转写器执行转写；文字稿任务跳过 STT
5. 调用 LLM 生成 Markdown
6. 注入关键时刻、时间戳与截图占位符；只有存在原始媒体时才会补充媒体预览和截图
7. 持久化任务产物并更新状态
8. 前端将最终笔记保存到数据库

## 总结模式

- `default`
  - 小内容走一次性总结，长内容自动升级为分层总结
- `accurate`
  - 总是先分块总结再合并
- `oneshot`
  - 总是一次性发送完整转写文本

## 用户体验上的结果

- 浏览器生成器支持 URL、本地音视频、本地文字稿三种入口
- 笔记标题可带时间戳跳转
- 关键时刻可以附带截图
- 预览侧媒体可跟随时间戳 seek
- 保存后的笔记可以生成公开只读分享链接
